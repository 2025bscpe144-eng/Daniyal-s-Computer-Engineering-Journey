The second semester project was significantly more ambitious than the first. Where semester one had us training a single regression model on a car dataset, the **Earthquake Prediction System** required a complete pipeline: data collection, database design, normalisation, ER diagram, model training, and evaluation. Each stage built on the previous one, and the quality of each stage affected the quality of what came after.

We sourced our dataset from **Kaggle** — a platform that hosts publicly available datasets used by data scientists and ML researchers worldwide. The earthquake dataset contained thousands of historical seismic events with features including magnitude, depth, latitude, longitude, date, and the recording station's data. It was rich, messy in places, and required significant preparation before any model could be trained on it.

The data preparation phase took longer than the model training itself, which is consistent with what **Dr. Bilal Ahmad** had taught us all semester. He emphasises that in any serious ML project — including the medical dataset work he conducts in his own research — data preparation consumes the majority of the time and effort. Cleaning outliers, handling missing values, encoding categorical features, and scaling numerical ones are not preliminary steps. They are the work. The model is almost the easy part.

```python
import pandas as pd
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import accuracy_score, classification_report

# Load cleaned dataset
df = pd.read_csv('earthquake_data_cleaned.csv')

# Feature selection
features = ['depth', 'latitude', 'longitude', 'gap', 'dmin', 'rms']
X = df[features]
y = df['magnitude_class']  # Classified as low / medium / high

# Scale features
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Train-test split
X_train, X_test, y_train, y_test = train_test_split(
    X_scaled, y, test_size=0.2, random_state=42
)

# Train Random Forest model
model = RandomForestClassifier(n_estimators=100, random_state=42)
model.fit(X_train, y_train)

# Evaluate
predictions = model.predict(X_test)
print(f"Accuracy: {accuracy_score(y_test, predictions) * 100:.2f}%")
print(classification_report(y_test, predictions))
```

We used a **Random Forest classifier** rather than simple linear regression because earthquake magnitude prediction is a classification problem with non-linear relationships between features. Random Forest builds multiple decision trees on subsets of the data and aggregates their predictions — making it more robust to noisy data and better at capturing complex feature interactions.

The model was trained on 80% of the data and tested on the remaining 20%, consistent with the split methodology Dr. Bilal Ahmad had established in semester one. Feature importance scores from the Random Forest revealed that depth and geographic coordinates were the strongest predictors of magnitude class — a finding that aligns with seismological theory, which was a satisfying validation.
